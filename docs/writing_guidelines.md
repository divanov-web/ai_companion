# Инструкции по написанию документации

Документация должна быть максимально сжатой и информативной.

### Основные принципы
- **Лаконичность**: минимум воды, максимум смысла.
- **Тезисность**: использование списков и коротких предложений.
- **Структура**: обязательное наличие заголовков и функционального описания.

### Требования к файлам
- Короткое описание назначения.
- Заголовки с тезисным описанием.
- Ссылки на связанные компоненты.
